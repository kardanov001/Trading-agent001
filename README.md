# Торговый агент — Инструкция по запуску

## Что это

Алгоритмический торговый агент для Т-Инвестиций. Работает по среднесрочной
трендовой стратегии на дневном графике. Реализует все правила из вашего промпта.

---

## 🚀 Текущий статус

Агент задеплоен на VPS и работает в автоматическом режиме:

- **Хостинг:** VPS (Linux)
- **Режим:** песочница T-Invest (виртуальные деньги)
- **Расписание:** автоматический запуск каждый будний день в 19:00 МСК через `cron`
- **Логи и отчёты:** сохраняются на сервере в `reports/daily/`, журнал сделок — в `reports/trades.csv`

Работает без вмешательства: получает данные с биржи, анализирует рынок и новости,
принимает решения по стратегии, выставляет заявки и формирует отчёт.

---

⚠️ **ВАЖНО:** это прототип для обучения и тестирования. Перед любым
использованием на реальные деньги — минимум 1-2 месяца тестов в песочнице.

## Шаг 1. Создать файл .env с токеном

В папке `trading-agent` создайте файл с именем `.env` (с точкой в начале).

В терминале Ubuntu:
```bash
cd ~/trading-agent
nano .env
```

Вставьте туда (замените на ваш токен):
```
TINKOFF_TOKEN=ваш_токен_от_тинькофф
TRADING_MODE=sandbox
```

Сохранить в nano: `Ctrl+O`, затем `Enter`, затем `Ctrl+X`.

⚠️ Файл `.env` НИКОМУ не показывайте — это доступ к вашему счёту.

---

## Шаг 2. Настроить счёт песочницы

Запустите команду один раз:
```bash
cd ~/trading-agent
source venv/bin/activate
python main.py --setup
```

Это создаст виртуальный счёт и пополнит его на 50 000 виртуальных рублей.

---

## Шаг 3. Запустить агента в режиме анализа (БЕЗ сделок)

```bash
python main.py --dry-run
```

В этом режиме агент:
- Проанализирует рынок
- Найдёт сигналы
- Покажет, что бы он купил
- НО НЕ выставит реальных заявок

Используйте этот режим первые несколько дней, чтобы понять логику агента.

---

## Шаг 4. Запустить агента в боевом режиме (песочница)

Когда убедились, что агент работает адекватно:
```bash
python main.py
```

Теперь агент будет выставлять заявки — но в песочнице, на виртуальные деньги.

---

## Шаг 5. Отчёты

После каждого запуска отчёты сохраняются в:
- `reports/daily/` — ежедневные отчёты
- `reports/trades.csv` — журнал всех сделок

---

## Автозапуск каждый день в 19:00

В Ubuntu (WSL2) используйте cron:

```bash
crontab -e
```

Добавьте строку (выберите редактор nano если спросит):
```
0 19 * * 1-5 cd ~/trading-agent && ./venv/bin/python main.py >> reports/cron.log 2>&1
```

Это запустит агента в 19:00 по будням (1-5 = пн-пт).

⚠️ Компьютер должен быть включён в это время, а WSL2 — запущен.

---

## Как менять стратегию

Все параметры — в файле `config.py`. Например:
- Изменить список бумаг → словарь `WATCHLIST`
- Изменить риск на сделку → `MAX_RISK_PER_TRADE`
- Изменить лимит позиций → `MAX_POSITIONS`

После изменений просто перезапустите агента.

---

## Если что-то не работает

1. Проверьте, что активировано venv: в начале строки терминала должно быть `(venv)`
2. Проверьте, что токен в `.env` правильный
3. Запустите с `--dry-run` чтобы увидеть, на каком шаге ошибка
4. Прочитайте сообщение об ошибке — там обычно написано, что не так

## ВАЖНЫЕ ПРЕДУПРЕЖДЕНИЯ

1. **Это прототип.** Реальная торговая система требует обработки сотен
   нештатных ситуаций (отказы биржи, дисконнекты, аномальные котировки).

2. **Логика выхода упрощена.** В прототипе шаг проверки открытых позиций
   (ШАГ 5) не полностью связан с журналом сделок. Это нужно доработать
   перед серьёзным использованием.

3. **Стратегия не протестирована на истории.** Никто не знает, прибыльна ли
   она. Обязательно проведите бэктест перед реальными деньгами.

4. **Песочница ≠ реальный рынок.** В песочнице нет проскальзываний,
   отказов заявок, частичных исполнений. Реальная торговля сложнее.

5. **Минимум 1-2 месяца в песочнице** перед переходом на реальный счёт.
   И начинайте с маленькой суммы (5 000 ₽), а не с 50 000 ₽.
